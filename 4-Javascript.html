JavaScript

Declaración de variables: let y const

let permite declarar variables que pueden ser reasignadas. const declara constantes que no pueden cambiar su valor después de la asignación inicial.

let contador = 0;
contador = 1; // Correcto, se puede reasignar

const PI = 3.1416;
PI = 3.14; // Error: no se puede reasignar una constante

Usa const por defecto para evitar cambios accidentales, y let solo cuando necesites reasignar.

Control de flujo

if-else

Ejecuta bloques de código basados en condiciones booleanas:

if (edad >= 18) {
	console.log('Mayor de edad');
} else {
	console.log('Menor de edad');
}

switch

Útil para múltiples casos basados en el valor de una variable:

switch (dia) {
	case 'lunes':
		console.log('Inicio de semana');
		break;
	case 'viernes':
		console.log('Fin de semana laboral');
		break;
	default:
		console.log('Día normal');
}

Bucles

while

Se ejecuta mientras la condición sea verdadera:

let i = 0;
while (i < 5) {
	console.log(i);
	i++;
}

for clásico

Bucle con inicialización, condición y actualización:

for (let i = 0; i < 5; i++) {
	console.log(i);
}

for-in

Itera sobre las propiedades enumerables de un objeto:

const persona = { nombre: 'Ana', edad: 25 };
for (let propiedad in persona) {
	console.log(propiedad + ': ' + persona[propiedad]);
}

for-of

Itera sobre elementos iterables como arrays:

const frutas = ['manzana', 'pera', 'uva'];
for (let fruta of frutas) {
	console.log(fruta);
}

forEach

Método de arrays para ejecutar una función en cada elemento:

frutas.forEach(function(fruta) {
	console.log(fruta);
});
// O con función flecha:
frutas.forEach(fruta => console.log(fruta));

Funciones

Función normal (declarada)

Se define con la palabra clave function:

function saludar(nombre) {
	return 'Hola, ' + nombre;
}
console.log(saludar('Mundo'));

Función anónima

Función sin nombre, asignada a una variable:

const saludar = function(nombre) {
	return 'Hola, ' + nombre;
};
console.log(saludar('Mundo'));

Función flecha

Sintaxis más concisa, especialmente para funciones simples:

const saludar = (nombre) => 'Hola, ' + nombre;
console.log(saludar('Mundo'));

// Para múltiples líneas:
const sumar = (a, b) => {
	const resultado = a + b;
	return resultado;
};

Esperar a que cargue el documento

Usar el evento DOMContentLoaded para ejecutar código cuando el DOM esté listo:

document.addEventListener('DOMContentLoaded', function() {
	// Aquí va el código que interactúa con el DOM
});

Esto evita errores al buscar elementos antes de que existan.

Obtener referencias al DOM

Formas habituales para obtener elementos del DOM:


		// Por id (devuelve un elemento directamente)
		const boton = document.getElementById('miBoton');

		// Por nombre de etiqueta (devuelve una colección similar a una lista)
		const forms = document.getElementsByTagName('form');
		const form = forms[0]; // primer formulario en la página

		// Por clase (devuelve una colección similar a una lista)
		const cajas = document.getElementsByClassName('caja');
		const primeraCaja = cajas[0];

getElementById devuelve un único objeto Element. getElementsByTagName y getElementsByClassName devuelven colecciones (HTMLCollection) que funcionan como listas: accede por posición (por ejemplo [0]) o itera la colección para procesar todos los elementos.

Añadir escuchadores

Formas sencillas para adjuntar listeners a elementos (ejemplos usando los métodos anteriores):


// Asignación directa (propiedades .onclick, .onsubmit, ...)
const boton = document.getElementById('miBoton');
const form = document.getElementsByTagName('form')[0];

boton.onclick = function() { console.log('click'); };
form.onsubmit = function(event) { event.preventDefault(); /* ... */ };

Pros: sencillo y rápido para casos simples. Contras: solo un handler por tipo de evento.


// Alternativa moderna: addEventListener (permite múltiples handlers)
boton.addEventListener('click', miFuncion);
boton.addEventListener('click', function() { console.log('otro handler'); });

form.addEventListener('submit', miSubmitHandler);

Pros: varios listeners, opciones de captura/bubbling y posibilidad de eliminar handlers con removeEventListener.

Funciones callback y el objeto evento

Ejemplo de callback:

function onClickBoton(event) {
	// El elemento que recibió el listener (el botón en este caso)
	console.log('currentTarget:', event.currentTarget);
}

boton.addEventListener('click', onClickBoton);

Para formularios, evitar que se envíen si queremos procesar datos en cliente:

function miSubmitHandler(event) {
	event.preventDefault(); // evita el envío
	// procesar datos aquí
}
form.addEventListener('submit', miSubmitHandler);

Crear elementos

Usando innerHTML

const cont = document.getElementById('contenedor');
cont.innerHTML = '<div class="tarjeta"><p>Hola</p></div>';

Usando createElement y appendChild

const elemento = document.createElement('div');
elemento.classList.add('tarjeta');
elemento.innerHTML = 'Hola';
const padre = document.getElementById('contenedor');
padre.appendChild(elemento);

La segunda opción es más segura y manipulable en tiempo de ejecución. Y permite añadir escuchadores al disponer del objeto.

Gestión de atributos

Forma genérica

const img = document.createElement('img');
img.setAttribute('src', 'foto.png');
console.log(img.getAttribute('src'));

Forma específica para Clases

const caja = document.getElementById('caja');
caja.classList.add('activo');
caja.classList.remove('activo');
caja.classList.toggle('visible');
// toggle: si no está activa, la activa; si está activa, la desactiva

Forma específica para datos personalizados - Dataset

const fila = document.createElement('div');
fila.dataset.idUsuario = 123; // se guarda como 'data-id-usuario'
console.log(fila.dataset.idUsuario);

Listas en JavaScript

// Creación
const lista = [];

// Añadir
lista.push('manzana');

// Acceso
console.log(lista[0]);

// Eliminar por valor
const index = lista.indexOf('manzana');
if (index >= 0) lista.splice(index, 1);

// Recorrer
for (const elemento of lista) {
	console.log(elemento);
}

También existen métodos funcionales como find, filter...

Ejemplo práctico: Añadir items a una lista

Mini-snippet para añadir elementos desde un formulario y mostrarlos:

document.addEventListener('DOMContentLoaded', function() {
	const form = document.createElement('form');
	const input = document.createElement('input');
	input.name = 'item';
	const submit = document.createElement('button');
	submit.type = 'submit';
	submit.textContent = 'Añadir';
	const ul = document.createElement('ul');

	form.appendChild(input);
	form.appendChild(submit);
	document.body.appendChild(form);
	document.body.appendChild(ul);

	const items = [];

	form.addEventListener('submit', function(e) {
		e.preventDefault();
		const form = e.currentTarget;
		const value = e.elements.item.value.trim();
		if (!value) return;
		items.push(value);
		const li = document.createElement('li');
		li.textContent = value;
		ul.appendChild(li);
		input.value = '';
	});
});

Objetos

Creación de objetos

Los objetos se crean usando llaves {} con pares clave-valor:

const persona = {
	nombre: 'Juan',
	edad: 30,
	ciudad: 'Madrid'
};

Acceso a propiedades

Se accede usando notación de punto o corchetes:

// Notación de punto
console.log(persona.nombre); // 'Juan'

// Notación de corchetes
console.log(persona['edad']); // 30

// Acceso a propiedad inexistente devuelve undefined
console.log(persona.pais); // undefined

Lectura y modificación

Puedes leer, modificar y añadir nuevas propiedades:

// Lectura
const nombre = persona.nombre; // 'Juan'

// Modificación
persona.edad = 31;
persona['ciudad'] = 'Barcelona';

// Añadir nueva propiedad
persona.email = 'juan@example.com';

// Eliminar propiedad
delete persona.ciudad;

Almacenamiento web: localStorage, sessionStorage y Cookies

Formas de almacenar datos en el navegador del cliente:

localStorage

Almacena datos sin fecha de expiración. Los datos persisten hasta que se eliminan explícitamente. Ideal para datos que deben mantenerse entre sesiones.

sessionStorage

Similar a localStorage, pero los datos se eliminan cuando se cierra la pestaña o ventana del navegador. Útil para datos temporales de una sesión.

Cookies

Pequeños fragmentos de datos enviados con cada solicitud HTTP. Pueden tener fechas de expiración y se usan comúnmente para autenticación o seguimiento. Tienen límites de tamaño (4KB) y se envían automáticamente al servidor.

localStorage y sessionStorage son APIs modernas y más convenientes para datos del lado del cliente, mientras que las cookies son más antiguas y se usan para comunicación cliente-servidor.

Guardar y recuperar listas de objetos en localStorage

Para almacenar estructuras de datos complejas como listas de objetos, usamos JSON.stringify para convertir a cadena y JSON.parse para recuperar:

// Crear una lista de objetos
const listaTareas = [
	{ id: 1, descripcion: 'Comprar leche', completada: false },
	{ id: 2, descripcion: 'Hacer ejercicio', completada: true }
];

// Guardar en localStorage
localStorage.setItem('tareas', JSON.stringify(listaTareas));

// Recuperar de localStorage
const tareasGuardadas = JSON.parse(localStorage.getItem('tareas'));
console.log(tareasGuardadas); // Muestra la lista original

Nota: Si la clave no existe, getItem devuelve null, por lo que es buena práctica verificar antes de usar JSON.parse.
